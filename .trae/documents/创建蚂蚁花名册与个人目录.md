## 目标
- 建立可追加的蚂蚁花名册（JSONL），记录 6 个 agent 的中英文名、等级经验、技能、工具。
- 在 agents 下为每个具名蚂蚁创建个人目录，且个人目录内固定包含 skills/ 与 tools/。
- 把现有 role_key 的“旧显示名”（情感工蚁/浏览器工蚁/文档工蚁 等）全部替换为你的“新名字”，让 UI/日志/向量库使用新名字。

- king：蚁王_特鲁 → King_Tru（Lv1 EXP0）

说明：emotion_worker/browser_worker/doc_worker 仍保留 role_key（用于代码路由与能力差异），但对外显示名不再出现“情感/浏览器/文档工蚁”这类旧称呼。

## 花名册（JSONL）设计
- 新增：`agents/ant_roster.jsonl`
- 每行一个 JSON 对象（英文 key，值含中英文）：
  - `id`：稳定唯一（如 `queen_sera`）
  - `role_key`：queen/king/soldier/emotion_worker/browser_worker/doc_worker
  - `name_cn` / `name_en`
  - `level` / `exp`
  - `skills`：数组，元素 `{ "slot": 1, "name": "技能名" }`
  - `tools`：数组，元素 `{ "slot": 1, "name": "工具名" }`
  - `tags`：数组，用于标注专长（例如 browser/doc/emotion），避免再用旧中文名

初始化时：skills/tools 先置为空数组，后续你随时追加技能/工具条目即可。

## 你问的“是否需要改其他地方引用/数据库能否清理重建”
- 需要改的“引用”主要是：`configs/agent_configs.yaml` 的 name（本计划会改）。role_key 不变，所以代码路由基本不需要动。
- 需要注意的“数据影响”：
  - 向量库 workspace_id 目前是 `ANT_USER_ID:agent_name`（见 chromaserver/service.py），agent_name 变了就会生成全新 workspace/collection。
  - 群聊历史也会按 name 记录展示。
- 你可以直接清理后重建：
  - 清空/删除 `chromaserver/data/chroma_vector_store` 与 `chromaserver/data/jsonl_storage` 后重新初始化即可重新生成。
  - 如你希望聊天历史也从 0 开始，可一并清理 `message/chatdata`（否则旧记录仍会存在但名字不匹配时展示会割裂）。
