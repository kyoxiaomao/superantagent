## 执行策略（保证文件级准确性）
### 1）生成“文件清单”并逐文件归类
- 覆盖范围：
  - `main.py`、`requirements.txt`、`.env.example`、`configs/**`、`agents/**`、`services/**`、`ui/**`、`chromaserver/**`、`memory/**`、`message/**`、`models/**`、`utils/**`。
- 排除范围（非代码文件/运行时生成物，避免噪音）：`logs/**`、`chromaserver/_logs/**`、`chromaserver/data/**`、`__pycache__/**`。
- 输出形态（DOCS 中落地）：
  - 按目录分节：每个目录一个小节。
  - 每个文件一行描述：
    - 文件职责（它做什么）
    - 文件内核心对象（主要类/函数/常量，按实际代码写）
    - 被谁调用/属于哪条运行链路（入口→…→该文件）

### 2）抽取“模块边界”并写模块化说明
- 模块划分（与仓库目录对齐，便于维护）：
  - 入口层：终端入口、UI 入口、向量服务入口
  - 运行时与编排：`services/*`
  - 智能体与工具：`agents/*`
  - UI：`ui/*`
  - 消息与落盘：`message/*`
  - 记忆：`memory/*` + `chromaserver/*`
  - 模型封装：`models/*`
  - 配置：`configs/*`
  - 工具目录：`utils/*`

### 3）为每个模块补齐“完整运行链路”
- 以“可复现的链路描述”为标准：写清楚入口命令 → 初始化步骤 → 关键对象创建 → 关键函数调用顺序 → 外部依赖（模型 API/向量服务） → 数据落盘位置。
- 核心链路至少包含：
  1. **UI 群聊链路**：`python -m ui.chat_app` → Tk 主线程渲染 → `AsyncColonyBridge` 后台 loop → `agentscope.init` → `create_colony` → `ColonyRuntime.start` → `UpdateScheduler/Heartbeat` → 向量库预热 → 用户输入 → `runtime.submit_user_text` → 蚁后推理 → 通过 AgentScope print 队列回流 UI。
  2. **终端交互链路**：`python main.py` → `agentscope.init` → `create_colony` → `ColonyRuntime.start` → stdin 输入 → `runtime.submit_user_text` → 蚁后回复。
  3. **向量库服务链路**：`python -m chromaserver.server` → HTTP 路由（`/health /info /init /call /shutdown`）→ `VectorStoreWorker.call` → `_ensure_ready` → `_build_reme_memory` → `ReMeUnifiedLongTermMemory` → Chroma 持久化目录与 JSONL 回退目录。
  4. **动态技能链路**：蚁王识别缺工具 → `services/workflow._ensure_tool_ready` → 兵蚁 `tool_create` → `services/skill_loader.safe_write_*` 写入 `agents/skills/<role_key>/` → `load_skills(register)` 动态注册到 Toolkit → 工蚁再次执行工具。
  5. **消息落盘链路**：群聊 `open_group_chat`/`MessageCenter.append_*` → 写入 `message/chatdata/*.jsonl`。
