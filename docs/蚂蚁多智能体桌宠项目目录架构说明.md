# 蚂蚁多智能体桌宠项目目录架构优化说明

本文以当前仓库代码为准，总结桌宠多智能体系统的目录架构，并为每个核心文件给出“代码层面的职责注释”，便于维护与扩展。

说明：本说明不展开以下目录：`.venv`、`.trae`、`__pycache__`、`docs`（除本文本身）、`logs`、`tests`。

## 1. 目录结构概览

## 2. 关键模块详细说明

### 2.1 入口与运行模式
*   **终端模式**：[main.py](file:///d:/antagent/main.py)
    *   `agentscope.init()` 初始化 AgentScope。
    *   通过 [services/model_manager.py](file:///d:/antagent/services/model_manager.py) 加载模型配置并构造 ModelBundle。
    *   通过 [services/workflow.py](file:///d:/antagent/services/workflow.py) 组装 colony。
    *   启动 [services/runtime.py](file:///d:/antagent/services/runtime.py) 常驻运行时并读取 stdin。
*   **UI 模式**：[ui/chat_app.py](file:///d:/antagent/ui/chat_app.py)
    *   Tk 主线程负责渲染与输入；后台线程在 asyncio loop 中运行常驻 runtime（见 [ui/async_bridge.py](file:///d:/antagent/ui/async_bridge.py)）。
*   **单轮执行器**：[services/workflow.py](file:///d:/antagent/services/workflow.py)
    *   适用于“给定输入→执行一次→返回输出”的调用方式，与常驻 runtime 并存。

### 2.2 agents/ (智能体层)
*   **统一创建入口**：[agents/base/ant_agent_base.py](file:///d:/antagent/agents/base/ant_agent_base.py)
    *   负责从 YAML 加载角色参数与 prompt，并创建 `ReActAgent`。
    *   负责把短期/长期记忆与 hooks（pre_observe/pre_reply/post_reply/pre_print）统一挂载到 agent，实现全链路结构化日志记录。
*   **蚁王**：[agents/king/agent.py](file:///d:/antagent/agents/king/agent.py)
    *   面向用户对话与任务调度；按 prompt 约束输出调度 JSON，供运行时解析并分发。
*   **兵蚁**：[agents/soldier/agent.py](file:///d:/antagent/agents/soldier/agent.py)
    *   提供“保存技能脚本/保存技能文档/列出技能工件”等工具，为动态技能闭环提供落地能力。
*   **工蚁（浏览器/文档/情感）**：[agents/worker/](file:///d:/antagent/agents/worker)
    *   浏览器工蚁与文档工蚁会在运行时加载 `agents/skills/` 下的动态技能，并提供 reload/list 工具。
    *   情感工蚁专注陪伴式回应，不注册工具。
*   **蚁后**：[agents/queen/](file:///d:/antagent/agents/queen)
    *   当前为占位目录，尚未加入 colony；后续可用于扩展与自修复能力。

### 2.3 services/ (服务与编排层)
*   **常驻运行时**：[services/runtime.py](file:///d:/antagent/services/runtime.py)
    *   **核心职责**：维护 MsgHub 群聊、接收用户输入、按心跳调度各角色发言、执行蚁王调度（dispatch/tool_create）闭环。
    *   **心跳机制**：默认 5 秒 tick，触发“群聊增量 → 长期记忆”同步。
    *   **话题机制**：当群聊空闲时，基于轮转顺序（King->Soldier->Worker...）与冷却时间，驱动智能体进行“发起话题/参与讨论/保持沉默”的决策；支持流式输出解析。
    *   **UI 事件**：可选将 agent 状态（busy/idle）、心跳、错误等转为结构化事件上报给 UI。
    *   **技能闭环**：检测工具缺失异常，驱动蚁王产出工具生成指令→兵蚁写入技能→工蚁 reload 生效。
*   **启动清理**：[services/startup_cleanup.py](file:///d:/antagent/services/startup_cleanup.py)
    *   通过环境变量 `ANT_RESET_ON_START=true` 触发。
    *   负责清除上一轮的运行时数据：长期记忆（memory/storage/*.jsonl）、动态技能（agents/skills/*.{py,md}）、生成的文档（docs/generated/）、事件日志（logs/）。
*   **colony 组装与单轮执行**：[services/workflow.py](file:///d:/antagent/services/workflow.py)
    *   组装参与者集合（蚁王/兵蚁/多工蚁），并提供单轮 `run_turn()` 执行路径。
*   **模型管理与适配**：[services/model_manager.py](file:///d:/antagent/services/model_manager.py)、[services/glm_chat_model.py](file:///d:/antagent/services/glm_chat_model.py)
    *   `model_manager.py` 读取 YAML 构造 ModelBundle。
    *   `glm_chat_model.py` 将 GLM HTTP API 适配为 AgentScope `ChatModelBase`。
*   **技能加载与安全写入**：[services/skill_loader.py](file:///d:/antagent/services/skill_loader.py)
    *   将兵蚁产出的脚本/文档安全写入 `agents/skills/`，并动态 import 调用 `register(toolkit)`。
*   **结构化事件日志**：[services/event_logger.py](file:///d:/antagent/services/event_logger.py)
    *   输出 JSONL 事件流，覆盖心跳/消息/工具调用/agent 状态，便于回放与排障。

### 2.4 configs/ (配置层)
*   **global_config.yaml**：AgentScope 初始化参数（日志级别、Studio URL 等）。
*   **model_configs.yaml**：GLM 适配参数（base_url/model/api_key 等），支持 `${ENV:-default}` 形式的环境变量替换。
*   **agent_configs.yaml**：角色 name/max_iters 与心跳参数（运行时读取用于调度）。
*   **prompts/system_prompts.yaml**：各角色 system prompt，尤其约束蚁王调度输出格式。

### 2.5 启动清理（便于测试）
*   通过环境变量 `ANT_RESET_ON_START=true` 开启启动清理；每次启动会删除上一轮动态产生的数据（长期记忆 JSONL、动态技能脚本/文档、生成文档、部分日志）。
*   实现位置：[services/startup_cleanup.py](file:///d:/antagent/services/startup_cleanup.py)，入口在 [main.py](file:///d:/antagent/main.py) 与 [ui/async_bridge.py](file:///d:/antagent/ui/async_bridge.py)。

### 2.6 message/ 与 memory/（运行时协议与持久化）
*   **message/**：统一 Msg 构造与解析，提供“从文本中提取 JSON”能力，供调度链路使用。
*   **memory/long_term_memory.py**：JSONL 长期记忆落盘与检索。
*   **memory/storage/**：每个 agent 一份 JSONL 数据文件；属于运行时数据，通常不建议提交。

## 3. 开发规范建议
1.  **引入规范**: 尽量使用绝对导入，如 `from agents.king.agent import KingAntAgent`。
2.  **消息传递**: 系统内部统一使用 `agentscope.message.Msg` 对象传递信息。
3.  **工具定义**: 动态技能脚本建议提供 `register(toolkit)` 入口，并为工具函数补充清晰的中文 docstring，便于自动生成 Schema 与调试。
