# 蚂蚁多智能体桌宠产品文档

## 一、产品概述

本产品是基于 AgentScope 框架（当前虚拟环境版本：agentscope 1.0.14，Requires-Python：>=3.10）开发的桌面多智能体助手，以蚂蚁族群为设定，通过蚁王、工蚁、兵蚁、蚁后四类不同职责的智能体协同工作，实现用户请求调度、桌面任务执行、多模态特殊场景处理、系统自主拓展等能力，将 AgentScope 的多智能体编排、工具调用、记忆与可观测性能力与桌面交互场景结合，打造可持续进化的趣味桌面助手体验。

## 二、产品定位

- **用户定位**：需要桌面管理辅助、喜欢趣味桌面工具的个人用户
- **功能定位**：集桌面管理、信息服务、安全监控、趣味交互为一体的多智能体桌面助手
- **技术定位**：基于 AgentScope 框架实现多智能体编排与自主拓展，轻量化桌面应用，支持跨平台部署，具备持续演进能力

## 三、核心功能

### 1. 用户交互与智能体编排（蚁王智能体负责）
由蚁王智能体（KingAntAgent）承接用户交互与多智能体调度编排。为保证系统一致性与可组合性，建议统一使用 `agentscope.message.Msg` 作为系统内部消息载体，并通过 `agentscope.pipeline` 的编排原语完成消息流转：

- 输入统一：把语音/文本/多模态输入统一封装为 `Msg(name="user", content=..., role="user")`
- 任务路由：解析任务意图与复杂度后，将任务分发给工蚁/兵蚁/蚁后
- 编排落地：
  - 组内协作与共享上下文：`MsgHub`
  - 串行链路（上一输出作为下一输入）：`sequential_pipeline`
  - 并行分发（同一输入并发给多个智能体）：`fanout_pipeline`
- 结果回传：把执行进度与最终结果统一转为消息回传，便于 UI、日志与追踪保持同一语义

文档中常用的“分层 / 对等 / 协调者-执行者”建议作为产品的协作拓扑描述；在 AgentScope 落地时分别对应：分层（Coordinator + `sequential_pipeline`）、对等（`MsgHub` 群聊）、协调者-执行者（Coordinator + `fanout_pipeline` + 汇总）。

### 2. 任务规划与协同执行（工蚁智能体负责）

由工蚁智能体（WorkerAntAgent）负责接收蚁王分发任务，完成“推理-行动-反馈”的闭环执行，建议优先使用 `agentscope.agent.ReActAgent` 以减少重复造轮子：

- 多步任务：通过 `ReActAgent(..., max_iters=...)` 控制推理-行动循环的最大迭代次数
- 工具执行：通过 `agentscope.tool.Toolkit` 注册桌面能力（文件整理、窗口管理、提醒等），用 `register_tool_function` 暴露为工具
- 并行工具调用：当模型生成多个工具调用时，可启用 `parallel_tool_calls=True`（按任务安全策略决定）
- 多工蚁协作：当前版本不存在 `Environment`；推荐用 `fanout_pipeline` 并行分工，用 `MsgHub` 共享上下文与互相 observe
- 上下文治理：短期记忆默认使用 `InMemoryMemory`；对话增长或长期运行建议开启 `compression_config` 控制记忆压缩，降低上下文膨胀风险

### 3. 多模态与安全监控（兵蚁智能体负责）

由兵蚁智能体（SoldierAntAgent）负责多模态与安全监控相关场景。AgentScope 的落地建议遵循“消息结构统一 + 介入点明确”：

- 多模态消息：统一用 `Msg` 承载，`content` 支持多模态内容块，保证在多智能体之间可传递、可复现
- 模型适配：通过 `agentscope.model` 的具体模型类接入具备多模态能力的模型（以所选模型实际能力为准）
- 实时介入：通过 `AgentBase` 钩子（`pre_observe/post_observe/pre_reply/post_reply` 等）实现风险检测与拦截；必要时调用 `interrupt()` 终止当前回复并进入自定义处理
- 可追溯：将风险事件、工具调用与关键决策写入日志/追踪，便于复盘与回归

### 4. 系统自主拓展与修复（蚁后智能体负责）

由蚁后智能体（QueenAntAgent）负责系统能力扩展与自修复，原则是最大化复用 AgentScope 原生能力、减少自定义封装：

- 工具扩展：通过 `Toolkit.register_tool_function` 将新增能力纳入工具系统；需要动态启停时使用工具组与 `reset_equipped_tools`，并在 `ReActAgent(..., enable_meta_tool=True)` 下由智能体自主管理工具集
- 外部工具接入：优先通过 `agentscope.mcp` 以标准协议接入远程工具，并通过 `Toolkit.register_mcp_client` 注册
- 自修复闭环：结合日志与追踪数据定位异常根因，再通过“受控工具调用/受控脚本执行”完成修复与验证
- 全局技能清单：将“skill 清单”定位为产品侧资产（配置/数据），由蚁后维护并持久化；在框架侧映射为 Toolkit 的工具组与可用工具集合

### 5. 趣味桌面交互

- 桌宠以 Tkinter 窗口展示，Pillow 负责图片/帧动画渲染，支持 idle/工作/移动等动画状态
- UI 事件（点击/拖拽/输入）统一转换为 `Msg` 投递到蚁王智能体，保证交互链路一致
- 与智能体输出的流式对接：可使用 `AgentBase.set_msg_queue_enabled` 与 `agentscope.pipeline.stream_printing_messages` 将中间打印消息以队列形式推送到 UI，实现平滑渲染与实时反馈

## 四、技术架构

### 1. 核心对象与数据流

- **消息（Msg）**：`agentscope.message.Msg` 是跨智能体与编排的统一载体（`name`/`role`/`content`/`metadata`）
- **智能体（AgentBase/ReActAgent）**：`AgentBase` 提供 observe/reply/interrupt 与钩子；`ReActAgent` 提供默认推理-行动闭环
- **工具系统（Toolkit）**：`agentscope.tool.Toolkit` 负责工具注册、工具组管理、MCP 工具接入
- **编排（Pipeline/MsgHub）**：`sequential_pipeline`/`fanout_pipeline` 覆盖串行与并行，`MsgHub` 覆盖群聊协作与共享上下文
- **记忆（Memory/LongTermMemory/Compression）**：短期对话记忆、长期记忆与压缩配置共同控制上下文规模与稳定性
- **检索增强（RAG）**：通过 `agentscope.rag.KnowledgeBase` 为回复注入外部知识，降低幻觉与漂移
- **初始化与可观测（init/Tracing/EventLogger）**：
  - `agentscope.init`：统一配置 Studio 调试与追踪。
  - `EventLogger`：全链路结构化日志（`services/event_logger.py`），记录心跳、消息、工具调用与 Agent 状态变化，生成 `.jsonl` 事件流，便于回放与分析。
  - `StartupCleanup`：支持开发模式下的“环境重置”（`ANT_RESET_ON_START=true`），自动清理旧的记忆与生成文件。

### 2. 心跳与话题自发机制（常驻运行时）
常驻运行时（`services/runtime.py`）提供“低频大模型调用 + 高频状态/记忆同步”的自发对话能力：

- **5 秒心跳 tick**：每个智能体周期性把群聊新增消息增量写入自己的长期记忆（每个 agent 一份 JSONL 文件），不产生额外发言。
- **话题轮转机制**：
  - **空闲探测**：当群聊持续超过阈值时间（默认 60 秒）没有新增消息时，触发话题决策逻辑。
  - **轮转顺序**：按照 `King -> Soldier -> Worker...` 的固定顺序轮流获得“话题发言权”，避免多智能体同时抢话。
  - **决策模型**：获得发言权的智能体通过独立的 prompt 进行思考，输出 JSON 决策：
    - `init_topic`: 发起新话题（需带 `#话题标签`）。
    - `contribute`: 参与当前话题。
    - `silent`: 保持沉默（如无话可说或话题不相关）。
  - **流式兼容**：支持对大模型流式输出的实时解析，确保决策指令的快速执行。
- **UI 事件驱动**：运行时将智能体状态（Busy/Idle）、心跳事件、错误堆栈等转化为结构化事件，供前端 UI 实时展示。

### 2. 推荐编排范式（效率优先）

#### 2.1 串行任务（分层/流水线）

```python
from agentscope.message import Msg
from agentscope.pipeline import sequential_pipeline

msg_in = Msg(name="user", content="整理桌面并设置 18:00 提醒", role="user")
msg_out = await sequential_pipeline([yi_wang, gong_yi, yi_wang], msg_in)
```

#### 2.2 并行子任务（多工蚁并发）

```python
from agentscope.message import Msg
from agentscope.pipeline import fanout_pipeline

msg_in = Msg(name="user", content="分别检查桌面文件、窗口状态、今日待办", role="user")
results = await fanout_pipeline([gong_yi_a, gong_yi_b, gong_yi_c], msg_in, enable_gather=True)
```

#### 2.3 群聊协作（对等协作/共享上下文）

```python
from agentscope.pipeline import MsgHub

async with MsgHub(participants=[yi_wang, gong_yi, bing_yi, yi_hou]):
    await yi_wang()
    await gong_yi()
```

#### 2.4 UI 流式输出对接（消息队列）

```python
import asyncio
from agentscope.message import Msg
from agentscope.pipeline import sequential_pipeline, stream_printing_messages

queue = asyncio.Queue()
coroutine = sequential_pipeline([yi_wang, gong_yi, yi_wang], Msg("user", "开始执行", "user"))
async for msg, is_last_chunk in stream_printing_messages([yi_wang, gong_yi], coroutine, queue=queue):
    _ = (msg, is_last_chunk)
```

### 3. 框架能力落地清单（按模块）

- `agentscope.init`：建议在 `main.py` 启动入口统一初始化项目名、运行名、日志路径、Studio 地址与 Tracing 地址
- `agentscope.message`：统一消息结构（文本/多模态/工具调用结果）
- `agentscope.agent`：
  - `AgentBase`：钩子、interrupt、observe、打印消息队列
  - `ReActAgent`：默认推理-行动闭环（`max_iters`、`parallel_tool_calls`、`compression_config`、`knowledge`、`plan_notebook`）
- `agentscope.tool`：`Toolkit`（注册工具、工具组、元工具、MCP 客户端）
- `agentscope.pipeline`：`MsgHub`（共享上下文/自动广播）、`sequential_pipeline`（串行）、`fanout_pipeline`（并行）、`stream_printing_messages`（输出流对接 UI）
- `agentscope.memory`：短期记忆与长期记忆；配合压缩控制上下文体量
- `agentscope.rag`：知识库检索增强（将桌宠的“常识/规则/用户偏好”结构化为可检索知识）
- `agentscope.hooks`：将安全监控、日志埋点、UI 事件回放等逻辑挂载到可控的生命周期节点
- `agentscope.mcp`：远程工具与服务接入的标准化通道

### 4. 项目架构

参考项目文件目录架构，分为智能体模块、UI 模块、工具模块、服务模块：

- **智能体模块（agents/）**：四个角色的智能体独立封装，建议基于统一的智能体基类与消息规范开发
- **UI 模块（ui/）**：桌宠可视化展示与交互（Tkinter + Pillow）
- **工具模块（utils/）**：桌面操作、图像处理、系统信息采集等工具函数集合（注册到 Toolkit）
- **服务模块（services/）**：任务分发、编排封装、模型配置加载、状态持久化等

## 五、智能体角色说明

|智能体角色|对应模块|核心职责|基于 AgentScope 的能力实现|
|---|---|---|---|
|蚁王智能体（KingAntAgent）|agents/KingAntAgent/|用户请求接收与智能体编排|以 `Msg` 统一输入输出，使用 `MsgHub` 共享上下文，用 `sequential_pipeline`/`fanout_pipeline` 实现串行/并行编排，作为协调者汇总并回传|
|工蚁智能体（WorkerAntAgent）|agents/WorkerAntAgent/|任务规划与协同执行|优先使用 `ReActAgent` + `Toolkit` 实现推理-行动闭环；并发用 `fanout_pipeline`；共享状态用 `MsgHub`|
|兵蚁智能体（SoldierAntAgent）|agents/SoldierAntAgent/|多模态与安全监控|多模态输入输出以 `Msg` 内容块承载；模型由 `agentscope.model` 适配；用钩子与 `interrupt()` 实现实时拦截；用 logging/tracing 做溯源|
|蚁后智能体（QueenAntAgent）|agents/QueenAntAgent/|系统拓展与自修复|以 `Toolkit` 为工具能力中心，通过工具组与 `reset_equipped_tools` 动态管理能力；通过 `agentscope.mcp` 接入远程工具；用日志与追踪数据驱动修复与回归|

## 六、开发环境要求

### 1. 依赖包

- Python 3.10+
- AgentScope（建议锁定版本：agentscope==1.0.14 或同系列版本）
- Tkinter（Python 标准库自带）
- Pillow（图像处理）
- 第三方 API（可选，敏感信息通过环境变量注入）

### 2. 开发工具

- IDE：TRAE（或 PyCharm / VS Code）
- 版本管理：Git
- 配置工具：YAML 编辑器

## 七、部署说明

1. 克隆项目代码到本地
2. 创建虚拟环境并安装依赖：`pip install -r requirements.txt`
3. 配置 `config.yaml`，填写模型配置、智能体配置等信息
4. 在启动入口调用 `agentscope.init(project, name, run_id, logging_path, studio_url, tracing_url)` 统一初始化日志与可视化调试能力
5. 运行 `main.py` 启动桌面桌宠

示例初始化（放在 `main.py` 的入口处）：

```python
import agentscope

agentscope.init(
    project="蚂蚁桌宠",
    name="本机运行",
    logging_level="INFO",
)
```
